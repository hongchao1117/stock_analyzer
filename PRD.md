## 1. 产品定位

### 一句话

> **AI Agent 自主分析任意股票/ETF（A股/美股/港股），实时获取行情，多角度挖掘新闻，生成操作建议报告。**

### 核心能力

| 能力 | 说明 |
|------|------|
| **自然语言输入** | 支持代码（`600519 AAPL`）或自然语言（`我持仓纳指科技ETF和SOXL`） |
| **AI Agent 自主决策** | Agent 自己决定分析角度、深挖程度、何时完成——而非固定流水线 |
| **多市场实时行情** | 新浪财经 API，覆盖 A股/美股/港股，零认证 |
| **多角度新闻分析** | Agent 根据涨跌幅和主题，从财报、政策、行业等角度分别生成新闻分析 |
| **可操作建议** | 综合行情 + 消息面，给出持有/关注/加仓/减仓 + 置信度 + 风险提示 |

### 产品形态

- **V1（当前）**：命令行交互 + 终端 rich 输出 + Markdown 文件
- **V2（远期）**：Web 面板，实时查看 Agent 决策过程
- **V3（远期）**：定时任务 + 推送通知

### 与固定流水线的本质区别

| 维度 | 传统 Pipeline | AI Agent（当前） |
|------|-------------|-----------------|
| 控制流 | 硬编码 `parse → fetch → analyze → output` | Agent 自主 Think → Act → Observe 循环 |
| 分析深度 | 每只股票固定分析 | Agent 判断：波动大→深挖、消息矛盾→多角度验证 |
| 输入 | 严格代码格式 | 自然语言 + 代码混合 |
| 容错 | 固定降级路径 | Agent 可换数据源重试、跳过失败标的 |
| 可解释性 | 每步固定可追踪 | 每步工具调用记录在 `tool_log` 中 |

## 2. 用户画像与场景

### 主用户画像：多市场个人投资者

| 属性 | 描述 |
|------|------|
| **身份** | 持有 A股 + 美股 + 港股中多只股票/ETF 的个人投资者 |
| **持仓特征** | 跨市场、关注特定产业链或主题 |
| **信息习惯** | 每天盘后花3分钟看汇总，不想逐个市场翻新闻 |
| **决策风格** | 中长期持有为主，关注单日大波动是否需要调仓 |
| **痛点** | 跨市场信息分散、不知道哪条新闻真正影响持仓、缺乏系统化每日检查清单 |

### 典型使用场景

```
场景1: 代码输入
  用户: python main.py
  👉 600519 AAPL 0700.HK
  👉 关注方向: 消费复苏
  → Agent 分析3只股票，重点从消费复苏角度切入

场景2: 自然语言输入
  用户: python main.py
  👉 我目前的持仓是纳指科技etf，中韩半导体ETF，美股soxl
  👉 关注方向: AI芯片
  → Agent 先调用 parse_portfolio 解析持仓
  → 然后分别从 AI芯片 角度分析3只标的

场景3: CLI 快捷模式
  用户: python main.py -s AAPL NVDA 600519 -t "AI产业链"
  → 跳过交互，直接输出报告
```

## 3. 核心用户旅程

### Agent 自主分析流程

```
用户输入（代码或自然语言）
    │
    ▼
┌─────────────────────────────────────────────┐
│  Agent 循环 (Think → Act → Observe)          │
│                                              │
│  Step 0: parse_portfolio                     │
│    "我持仓纳指科技ETF和SOXL" → 513100, SOXL  │
│                                              │
│  Step 1: fetch_market_quotes                 │
│    新浪财经 → 实时价格/涨跌幅/成交量         │
│                                              │
│  Step 2-4: analyze_stock_news × N            │
│    Agent 根据涨跌幅选择分析角度:              │
│    - 大涨/大跌 → "财报", "行业政策"          │
│    - 用户主题 → 从该主题切入                 │
│    - 普通波动 → "行业动态", "技术面"         │
│                                              │
│  Step 5-7: generate_trading_advice × N       │
│    综合行情 + 所有新闻 → 操作建议            │
│                                              │
│  Step 8: submit_final_report                 │
│    Agent 确认分析完毕，提交报告              │
└─────────────────────────────────────────────┘
    │
    ▼
  终端 rich 输出 + output/report.md
```

### Agent 自适应示例

```
茅台 (600519) 今日跌 2.02%:

  无主题时:
    新闻角度 → 消费税改革、端午消费疲软、直销渠道
    消息面 → 偏空 (1利好/2利空)
    建议 → 📉 减仓（高置信度）

  主题 "AI数字化转型" 时:
    新闻角度 → AI智能酿造系统、华为合作大模型、数字化营销
    消息面 → 偏多 (2利好/1利空)
    建议 → 📈 加仓（中置信度）—— 跌了反而是抄底机会
```

## 5. 数据模型

### 5.1 核心数据结构

```python
StockInfo        — 解析后的股票信息（代码、市场、名称）
MarketSnapshot   — 实时行情快照（价格、涨跌幅、成交量、市场状态）
NewsItem         — 分析后的新闻（标题、情感、重要度、来源、可点击链接）
StockAdvice      — 操作建议（动作、置信度、理由、风险提示）
DailyBriefing    — 每日简报（汇总所有快照、新闻、建议）
AgentState       — Agent 运行时状态（步数、工具日志、累积数据）
```

### 5.2 Agent 工具集

| 工具 | 用途 | 调用时机 |
|------|------|---------|
| `parse_portfolio` | 自然语言 → 股票代码 | 用户输入不是纯代码时 |
| `fetch_market_quotes` | 获取实时行情 | 分析开始 |
| `analyze_stock_news` | 生成/分析股票新闻 | 每只股票1-N次（Agent决定角度和次数） |
| `generate_trading_advice` | 生成操作建议 | 每只股票新闻分析完成后 |
| `submit_final_report` | 提交最终报告 | 所有分析完成 |

## 6. 架构设计

### 架构原则

1. **Agent 自主决策 > 硬编码流程** — Agent 决定何时调用哪个工具
2. **工具可组合** — 每个工具独立可测试，Agent 负责编排
3. **失败不崩溃** — 工具执行异常返回错误信息，Agent 决定如何应对
4. **过程可审计** — 每一步工具调用记录在 `tool_log` 中

### 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| Agent 引擎 | DeepSeek Chat API + function calling | 决策 + 工具编排 |
| 行情数据 | 新浪财经 HTTP API | A股/美股/港股实时数据 |
| 终端渲染 | rich | 彩色表格、面板 |
| 文件输出 | Markdown | report.md |
| 运行时 | Python 3.11+ | requests, pyyaml |

## 7. 风险登记册

| # | 风险 | 缓解措施 |
|---|------|---------|
| R1 | DeepSeek API 不可用 | 工具调用失败返回 error，Agent 可跳过或降级 |
| R2 | 新浪财经数据延迟 | 标注数据时间，Agent 可根据时间判断有效性 |
| R3 | Agent 循环不终止 | max_steps=8 强制截断 |
| R4 | 自然语言解析错误 | parse_portfolio 返回 unrecognized 列表，用户可纠正 |
| R5 | LLM 生成新闻不够准确 | 标注"AI生成，仅供参考"，附百度搜索链接供验证 |

## 8. 成功标准

| # | 标准 | 验证方式 |
